- 태그 란?
HTML 문서(웹 서비스를 하기 위한 기본적인 문서)를 구성하고 있는 요소, 태그에는 이름과 속성이 있다.
/ 가 붙어있는 태그 : 열린 태그(시작 태그)
/ 가 붙어있지 않은 태그 : 닫힌 태그(종료 태그)
시작 태그와 종료 태그는 항상 쌍을 이룬다.
<img src = "이미지 명"> - img : 태그 이름, src : 태그 속성, 이미지 명 : 속성 값
HTML 버전은 HTML5 를 활용한다.
HTML5 는 어떤 웹 사이트를 서로 다른 브라우저에서 볼 때 약간의 차이가 존재하는 것을 막기 위해 웹 표준에 대한 부분을 많이 지키고 있다.

<!DOCTYPE html> <!--문서의 타입을 지정(html 파일)-->

<head><!--문서의 정보 및 타이틀을 지정하는 태그-->
    <!--스타일 시트의 내용, 자바스크립트의 내용과 같은 환경설정에 관한 정보들이 존재할 수 있다.-->

<body>
    <!--전달하고자 하는 정보들이 들어가는 태그-->

<br /> <!--개행 태그, 종료태그(/)와 시작태그(br)가 한 몸으로 되어 있다.-->
<hr /> <!--개행한 후 줄을 한 줄 그어서 구분을 해주는 태그-->

<h1>태그입니다.</h1><!--페이지의 타이틀이 될 태그, 숫자가 커지면 커질수록 글자의 크기가 작아진다, 최대 6까지 존재한다.-->
    <h2>태그입니다.</h2><!--개행 태그를 따로 넣어주지 않아도 한 단락을 이루고 있기 때문에 바로 개행이 되어진다.-->
    <h3>태그입니다.</h3>
    <h4>태그입니다.</h4>
    <h5>태그입니다.</h5>
    <h6>태그입니다.</h6>

<p>html 학습용 <br />p 태그</p><!--본문에 사용되는 태그, 하나의 문단이자 단락이 되는 태그이다.-->

<p><a href="http://www.google.com">구글 홈페이지</a></p><!--a 태그 : 다른 웹 사이트 하이퍼 링크 태그(앵크 태그), href 속성에 어떤 사이트로 갈지 경로를 지정해준다.(프로토콜이 있다면 프로토콜 까지 정확하게 기입할 것)-->

리스트 태그, 테이블 태그, 이미지 출력 태그, 사운드 출력 태그(HTML5 에서 추가된 기능 - 많이 쓰이는 태그는 아님)

리스트 태그 - ul, li, ol(리스트를 표현하는 가장 많이 사용되는 태그)
ul - unorder list type : 순서에 대한 인덱스 가 없음
ol - order list type : 순서에 대한 인덱스가 존재한다.
li - list item : 리스트를 나타내는 하나하나의 아이템들

리스트 구조의 태그가 중요한 이유는 웹 문서를 만들때 연속적인 덩어리(텍스트 이건 이미지이건)로 수직적이거나 수평적으로 나열된다면
css 로 예쁘게 웹 문서를 만들어 나갈수 있다.

a 태그 target 속성
_blank, _self
_blank : 링크 클릭 시 새 창이 열린다.
_self : 링크 클릭 시 현재 웹 페이지에서 해당 링크로 이동한다.

표를 나타내는 태그 - table
표형식의 레이아웃을 만드는 태그로, 예전에는 웹 사이트의 전체적인 레이아웃을 만들었으나 현재는 사용빈도가 많이 줄어들었다.
예전처럼 웹 문서의 전체적인 레이아웃을 잡는 용도로는 잘 사용하지 않는다. 나중에 유지보수를 하기 힘들고, 테이블 태그에 한계성이 많았기 때문
<!--table 태그로 시작-->
        <!--border 속성값 : 테이블을 구성하고 있는 레이아웃의 두께-->
        <!--tr : table 의 하나하나의 행을 나타내는 태그-->
        <!--td : 하나하나의 열을 나타내는 태그-->

테이블 태그를 사용하는 빈도는 많이 줄어들었으나 회원가입과 같은 폼의 경우는 아직 쓰이는 경우가 있기는 함
cell 을 합치는 rowspan, colspan 과 같은 속성들도 많이 쓰일것 같지만 실제로는 많이 쓰이지 않는다.
테이블을 합치다 보면 테이블이 깨지는 경우도 있고 의도치 않은 형태의 테이블이 만들어지는 경우도 있음

이미지를 나타내는 태그 - img 태그
웹 사이트에서 이미지를 출력해 주는 태그이다.
img 태그 이후 src 속성을 넣어준다.
src 는 이미지가 있는 경로를 표시해주는 속성이다.
경로를 표시해줄때 이미지가 있는 위치가 어떤 폴더 안에 들어가 있을 경우 / 를 사용하여 해당 경로를 정확하게 작성해주어야 한다.
alt 태그 : 이미지가 해당 경로에 정확하게 있으면 문제없이 출력되나, 해당 이미지가 경로상에 존재하지 않거나 네트워크 불안정으로 인해 이미지를 불러오지 못했을 경우를 나타내주는 태그
웹 접근성을 고려해서 웹 문서를 작성해야 하기에 이미지를 불러올땐 꼭 alt 태그를 함께 사용해준다.
alt 태그에 들어갈 문구는 특별히 정해져 있지 않으며 그냥 이미지를 표현할 적절한 문구를 넣어주면 된다.
src 에 이미지 경로를 표현할때 원격지에 있을 경우(웹 사이트 상에 해당 이미지의 경로가 있을 경우) - 해당 웹 사이트의 프로토콜까지 정확하게 입력해줘야 한다.
width 속성을 사용하지 않으면 출력되는 이미지의 크기는 해당 이미지가 원래 가지고 있는 이미지가 출력되나
width 속성을 사용하여 크기 값을 지정해주면 출력하고자 하는 이미지의 크기가 width 속성에서 지정해준 크기에 맞게 출력된다.
그러나 width 속성은 현재는 거의 보기 드물다.
현재 이미지의 크기는 css 에서 컨트롤한다.

사운드를 출력하는 태그 - audio
웹 사이트에서 사운드를 출력해주는 태그이다.
HTML5가 들어오면서 html 내부적으로 audio 태그가 생겼으므로 별도의 플러그인 없이 웹 페이지에서 사운드를 출력해주는 것이 가능해졌다.
src 속성에는 해당 사운드 파일이 정확하게 존재하는 경로를 입력해준다.
controls 속성은 넣어도 되고 넣지 않아도 된다.
controls 속성은 사운드의 컨트롤러를 출력 시킬지, 그러지 않을지를 결정하는 속성으로서 컨트롤러를 넣고 싶지 않다면 해당 속서을 기입하지 않으면 된다.
autoplay 속성은 웹 페이지에서 자동으로 해당 사운드 파일을 자동으로 재생시키고자 한다면 넣어도 되나 그렇지 않다면 빼버려도 상관없다.
loop 속성은 무한 반복 재생을 할지 한 번만 재생할지를 결정하는 속성이다.
